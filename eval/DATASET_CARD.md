# 평가 데이터셋 카드 (QA Dataset Card)

평가셋이 **어떤 기준·방법으로 만들어졌는지** 기록한다. RAGAS·IR 점수의 신뢰 범위를
판단하려면 이 문서를 함께 봐야 한다. 생성 코드: `eval/build_dataset.py`.

## 1. 소스 코퍼스
- `data/` 의 4개 PDF(시장·LGES·CATL·기술 배경) → PyMuPDF 파싱 → 500자/100overlap 청킹
  → BAAI/bge-m3 임베딩 → FAISS. 총 **262 청크**.
- 질문은 이 청크들에서 생성한다(코퍼스 내부 평가, closed-book 아님).

## 2. 샘플링 기준
- 길이 200자 미만 청크(목차·페이지번호 등 정보량 적은 것) 제외 → 후보 240개.
- **결정적 균등 샘플링**(stride = 240/n): 난수 미사용 → 재현 가능, easy/hard가 **동일 청크**를
  사용하므로 난이도 비교가 공정.
- 최초 기준선은 n=30으로 만들었고, 최종 해석은 표본 노이즈를 줄이기 위해 easy/hard 각각 n=100까지 확장한 결과를 우선 사용한다.

## 3. 생성 방법
- GPT-4o-mini(temperature=0)로 **청크 1개당 질문 1개 + 정답 1개** 생성.
- 모든 난이도 공통 규칙:
  - 해당 청크 **하나만으로** 답할 수 있어야 함(single-hop, 검증 가능성 확보).
  - 청크에 없는 내용 질문·정답 생성 금지(정답은 청크에 근거).
  - "이 문서/이 청크" 같은 **메타 질문 금지**(출처 모르고도 이해되는 자립 질문).
  - 정답은 1~3문장.

## 4. 난이도 기준 (핵심)

| 난이도 | 의도 | 작성 기준 |
|---|---|---|
| **easy** | 검색이 쉽게 성공하는 기준선 | 청크의 핵심 용어·수치·기업명을 질문에 **그대로** 사용한 직접 질문 |
| **hard** | 첫 검색이 실패할 여지를 만들어 **재검색(Agentic RAG)의 효과를 시험** | 아래 기법 1~2개 적용 |

**hard 난이도 기법**:
1. **동의어/상위어 치환** — 고유 용어를 일반어로 (예: "연결기준 매출"→"한 해 벌어들인 총수익").
2. **풀어쓰기(paraphrase)** — 청크 문장과 다른 어순·표현으로 재서술.
3. **간접/추론** — 키워드 직접 매칭이 아닌 한 단계 추론을 요구.
- 제약: 정답은 여전히 **그 청크 하나**에 근거(IR gold 검증 유지). 가장 식별력 높은
  키워드(고유명사·수치)는 질문에 그대로 노출하지 않음.
- 각 항목의 적용 기법은 `question_type`(synonym/paraphrase/inference)에 기록.

**왜 이렇게 나눴나**: Agentic RAG의 재검색은 "첫 검색이 부족할 때 복구"하는 장치다.
easy처럼 질문↔청크 어휘가 일치하면 첫 검색이 거의 성공해 재검색이 발동하지 않아 효과 측정이 불가능하다.
hard는 질문↔청크 사이에 **어휘 격차**를 의도적으로 만들어 재검색이 실제로 작동할 조건을 부여한다.
두 셋의 델타를 비교하면 "재검색이 어려운 질의에서 실제로 검색 품질을 회복하는가"를 가린다.

## 5. 정답(gold) 라벨링
- `gold_chunk` = 질문을 생성한 **원본 청크의 page_content 전체**.
- IR 평가는 검색 결과 청크의 page_content와 **정확 문자열 매칭**으로 정답 포함 여부·순위를 판정
  (같은 벡터스토어에서 나왔으므로 매칭 보장).
- `source`/`page`는 사람이 추적용으로만 사용.

## 6. 항목 스키마
```json
{
  "question": "질문",
  "ground_truth": "청크 근거 정답(1~3문장)",
  "difficulty": "easy | hard",
  "question_type": "direct | synonym | paraphrase | inference",
  "gold_chunk": "정답 청크 원문",
  "source": "원본 PDF 파일명",
  "page": 11
}
```

## 7. 한계 (정직)
- **합성 데이터**: 사람이 큐레이션한 질문이 아니라 LLM 생성. 실제 사용자 질의 분포와 다를 수 있음.
- **Single-hop**: 한 청크로 답하는 질문만 포함. 여러 청크를 종합해야 하는 multi-hop 질의는 미포함
  (IR gold 라벨이 단일 청크라 검증을 단순화하기 위함).
- **정답 근거는 생성 지시에 의존**: 별도 인적 검수 없이 temp=0 grounding 지시로만 보장.
- **표본 크기 민감도**: n=30에서는 ±0.03~0.09 수준 델타가 노이즈에 묻히는 현상이 확인되었다. 최종 결론은 n=100 기준으로 해석하되, n=100에서도 ±0.02~0.03의 소폭 차이는 단정하지 않는다.
- **난이도는 상대적**: hard도 결국 코퍼스 내부 질문 → 외부 도메인 일반화 보장 아님.

## Web Search 평가 토픽셋 (web_topics.json)

- 목적: 3방향 쿼리(긍정/비판/중립)의 확증편향 저감 효과 측정용 base 쿼리 모음.
- 구성: 총 20개. 프로덕션 실제 쿼리 3개(`prod_*`) + 배터리 기업전략 파생 토픽 17개.
- 카테고리: market / company / company_segment / technology / policy / comparison.
- 선정 기준: "기업 전략 자료가 긍정 편향되기 쉬운" 성격의 토픽(가설 적용 대상)으로 한정.
- 각 항목 `subject`는 stance/risk 판정의 대상 엔티티(기업·기술·시장).
- 한계: 배터리 산업 한정, 외부 도메인 일반화 보장 아님.
